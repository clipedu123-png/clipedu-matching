# 원생매칭 랜딩 페이지 — 종합 리뷰

> 평가 대상: `원생매칭 랜딩.html` (원본) — `원생매칭 랜딩 v2.html`은 후기 영역만 개선됨.
> 평가 관점: **랜딩 페이지의 전환(conversion) 설계** + **스캐닝 친화성**.
> 평가 일자 기준: 2026-05-26.

---

## 한 줄 요약

**신뢰감·정보량·시각 완성도는 좋지만, "랜딩 페이지로서의 전환 설계"와 "스캐닝 친화성"이 약함.**
방문자가 3초 안에 "이게 뭔지·나에게 무엇이 좋은지·다음에 뭘 해야 하는지"를 파악하기 어렵고, 핵심 정보가 아코디언과 긴 문단 안에 묻혀 있음.

## 1. 텍스트 밀도 (사용자가 짚은 문제 — 가장 큼)

### 1.1 가장 두꺼운 곳 Top 5

| 위치 | 문제 |
|---|---|
| **§01 About — 본문 3단락** (HTML 1317~) | 한 단락이 3~4줄, 시각적 break 없이 연속. 핵심 키워드 굵게 처리는 있지만 단락 자체가 길어 스킵됨. |
| **§02 어떻게? — 4 step descriptions** (HTML 1485~) | 각 step body가 2~3 문장 + 부연 설명. step 02는 "조건에 부합하는 학원…인입되면 다시 상담을 진행합니다"까지 4줄. |
| **§02 유입경로 — 4 flow + 설명** (HTML 1383~) | 대표/최근 두 그룹 + flow 4개 + 칩 텍스트 다수 → 정보량 과적재. |
| **§02 왜? — 3 reason descriptions** (HTML 1531~) | 각 reason이 한 문장으로 끝나야 할 것이 "유아~중학교 저학년…그만큼 검증된 학원으로 알아보길 원하십니다"처럼 다문장. |
| **§04 Contact 인트로** (HTML 1622) | "많은 학생(학부모님)들이 아직까지도 학원보다 신청자(학생·학부모님)가 더 많아서, 협력할 학원을 기다리고 있어요. …" — 한 문장에 다중 절. |

### 1.2 왜 이렇게 느껴지는가
- **단락이 단위**다. 한 단락 = 한 정보 덩어리 = 한 시각 블록인데, 한 단락에 정보를 2~3개씩 우겨넣음.
- **굵게(`<strong>`)·하이라이트가 본문 안**에 있어 단락을 깰 만큼의 시각 무게가 없음.
- **bullet · 카드 분리가 거의 없음**. 모두 prose(통문장)로 처리.
- **모바일 대응 불필요(데스크탑 1280)**임에도 가독 line-height가 1.75~1.9로 빼곡하지 않은 편이지만, 단락 자체 길이가 길어 무겁게 보임.

### 1.3 해결 원칙 (적용 시 적용 강도순)
1. **3-문장 규칙**: 한 단락 = 최대 3문장. 넘으면 단락을 쪼개거나 bullet으로 변환.
2. **첫 문장 굵게**: 각 단락의 첫 문장(혹은 핵심 명사구)만 `<strong>`로 처리. 스캔할 때 그 문장만 읽어도 메시지가 전달되도록.
3. **숫자/키워드 카드화**: "60,000+ 누적 매칭" 같은 수치는 단락에서 빼서 `.stat` / `.isum` 카드로.
4. **Process step body 2줄로 압축**: §02 어떻게? 의 step 설명을 "한 줄 요약 + 짧은 부연 1줄"로 강제.
5. **인용·예시 격리**: 본문에 섞인 예시("어디 학교 몇 학년인지…")는 `.panel-note` 박스나 별도 chip 그룹으로 분리.

### 1.4 카피 톤 인지 부담
- 모든 단락이 **합니다체**. 친근감·해소감이 부족해 더 무겁게 느껴짐.
- 마이크로 카피(라벨, 호버, 작은 안내)는 **해요체**로 풀면 호흡이 살아남.
  - 예: "신청서 남겨주시면 24시간 내로 더 정확하고 자세하게 상담해 드리겠습니다!" → "신청서 남겨주시면, 24시간 안에 더 정확히 상담해 드려요."

## 2. 랜딩 페이지로서 결정적으로 부족한 것

### 2.1 HERO에 CTA가 **없음** (가장 큰 문제)
- 첫 화면에 버튼이 0개. "SCROLL"이라는 스크롤 큐만 있음.
- 이 페이지는 "원장님이 학원에 원생을 받기 위해 신청하는" 페이지인데, 첫 화면에서 신청 동선이 없음.
- `.btn-solid` / `.btn-ghost` 스타일은 CSS에 이미 정의되어 있지만 HTML에 안 쓰임.

**조치**: HERO 본문 아래 `.hero-actions`로 **CTA 2개** 배치.
- Primary: "지금 매칭 신청하기" → `#contact`
- Secondary: "어떻게 운영되나요?" → `#projects` 또는 모달

### 2.2 NAV에 메뉴·CTA 0개
- 현재 nav에는 브랜드 로고만 있음. 메뉴, anchor 링크, 신청 버튼 모두 없음.
- 페이지가 길어 중간쯤에 도달하면 위로 돌아가거나 다른 섹션으로 가기 어려움.

**조치**:
- 데스크탑 fixed nav에 4개 메뉴(`서비스·작동방식·후기·문의`) + 우측 끝 작은 "매칭 신청" CTA(.nav-cta).
- 스크롤 다운 중에도 항상 신청 동선 확보.

### 2.3 가격/비용 정보 **0건**
- "원장님이 우리한테 얼마를 내야 하는가?"에 대한 정보가 페이지 어디에도 없음.
- B2B 랜딩의 핵심 질문 1번이 빠짐.

**조치**: 최소 정책 한 줄(`성과형 / 월정액 / 무료`) 또는 "비용 안내는 상담 시 안내" 명시 — 빈칸으로 두면 의심을 사거나 이탈함.

### 2.4 아코디언 남용 → 핵심 정보 은닉
- `§02 How It Works`의 4개 아코디언이 **전부 닫힌 상태로 시작**.
- 방문자의 70~80%는 아코디언을 열지 않음(웹 통계 일반화).
- "유입경로", "얼마나", "어떻게", "왜" — 이 4가지가 다 핵심인데 다 숨김.

**조치**: 적어도 **"어떻게?"(4-step 프로세스)**는 카드/타임라인으로 펼쳐서 노출. 부가 데이터(유입경로 디테일, FAQ 등)만 아코디언화.

### 2.5 청자 혼란 (academy vs parent)
- Hero: "원생 유지, 자신 있으신가요? 원생 유입은 우리가 할게요!" → **학원 원장님 대상**.
- §02 왜?: "**학부모님**들이 CLIPEDU를 선택하는 확실한 이유" → 학부모 대상으로 휙 전환.
- Contact form: "학원명/사업장 주소" → 다시 학원 대상.
- 한 페이지 안에서 청자가 학원↔학부모를 왕복하면 "이 서비스는 누구를 위한 거지?" 혼란.

**조치 옵션**:
- A. 페이지를 **학원용**으로 통일 → §02 "왜?"를 "왜 학원이 CLIPEDU를 선택하는가"로 재작성.
- B. 학부모용 정보는 별도 sub-페이지(또는 섹션 분리 + visual switcher)로 빼기.
- 현재 페이지는 학원 신청 폼이므로 A가 자연스러움.

### 2.6 사회 증명(social proof) 늦게 등장
- 후기(§03)가 5번째 섹션. 그 전에 지도·소개·작동방식이 와서, 신뢰 형성 전에 정보 부하부터 옴.
- 일반적 랜딩: HERO → 즉시 핵심 수치/로고 → 작동방식 → **후기** → CTA.

**조치**: 후기 영역 일부(또는 미니 버전)를 HERO 직후 또는 §01 About 안에 끼워넣기. 예) Hero 아래에 **"이미 480곳의 학원이 함께하고 있어요"** + 학원 로고 5~6개 띠.

## 3. 데이터·카피 일관성 문제 (신뢰도 직격)

### 3.1 숫자 mismatch
| 지표 | About에 표기 | Project §02에 표기 | 차이 |
|---|---|---|---|
| 누적 매칭 | **60,000+** | 12,400건 | ~5배 차이 |
| 제휴 기관 | **1,200+** | 480여 곳 | ~2.5배 차이 |
| 월 평균 문의 | 8,500+ (Inquiry) | **8,560건** (Project) | 60건 |

→ 같은 페이지 안에서 핵심 숫자가 다름. 검증 시 신뢰 무너짐. **한 곳을 단일 진실로 고정 필요.**

### 3.2 연도 표기
- Hero: `EST. 2026 — 원생매칭 플랫폼`
- Footer: `© 2025 CLIPEDU. All rights reserved.`
→ 2026년 설립인데 카피라이트는 2025? 한 쪽 오타.

### 3.3 자체 모순 카피
- §04 Contact 인트로: "학원보다 **신청자(학생·학부모님)가 더 많아서**, 협력할 학원을 기다리고 있어요"
- 이는 학원에게 **유리한 메시지** → 좋음.
- 하지만 §02 "얼마나?"에서 "480여 곳의 제휴 기관"이라는 비교적 작은 숫자와 같이 노출되면 "그럼 우리도 빨리 가입 안 하면 늦겠다"는 긴장감보다는 "아직 작은 서비스인가?" 인상.

### 3.4 사실/추정 마크업 부재
- 모든 수치가 동등한 무게로 표시됨. "추정" vs "확정" 구분 없음.
- 예: "98% 학부모 만족도" 같은 수치는 출처/조사 방법 footnote가 있어야 신뢰 ↑.

## 4. 정보 구조 (IA) 재배치 제안

### 4.1 현재 순서
```
HERO → Inquiry Map → About → How It Works → Reviews → Contact
```

### 4.2 권장 순서 (학원 대상 전환 목적)
```
HERO  (CTA 2개)
  ↓
[NEW] Quick proof bar  ← "480곳 학원과 운영 중" + 학원 로고
  ↓
About (3단락 → 1단락 + 3 카드)
  ↓
How It Works (현재 §02 어떻게?만 펼쳐서 노출)
  ↓
Reviews (현재 §03, v2 phone mockup)
  ↓
[NEW] Pricing or "비용 안내"
  ↓
[NEW] FAQ
  ↓
Inquiry Map (지역 확인하고 싶은 사람용 부가 정보)
  ↓
Contact / 신청 폼
  ↓
Footer
```

### 4.3 지도 섹션의 위치 재고
- Inquiry Map은 **시각적으로는 매력적**이지만 "원장 의사결정"에 직접 영향은 약함.
- 원장 입장에서 "내 지역에서 운영되나?" 확인 용도 → 페이지 하단·접힌 형태로도 충분.
- HERO 바로 다음에 두기엔 인지 부하가 큼.

## 6. 시각 디테일 보완

### 6.1 이미 정의됐지만 미사용된 컴포넌트
- `.btn-solid`, `.btn-ghost` — HERO·중간 CTA에 활용.
- `.hero-actions` — 위 두 버튼의 컨테이너.
- v2의 `.review-stats` / `.review-stat` — featured 안에 수치 강조 가능.

### 6.2 통계 카드(`.stat`) 활용 부족
- 현재 4개 stat이 §01 About 내부에 한 번만 등장.
- HERO 옆 또는 직후 별도 strip으로 노출하면 신뢰도 즉발.

### 6.3 시각 변주 부재
- 모든 섹션이 동일 구조(sec-num + sec-label + sec-title + content).
- 1개 섹션 정도는 다크 배경(이미 `#contact`가 다크지만 마지막이라 효과 약함) 또는 풀블리드 이미지 섹션으로 리듬 변화.

### 6.4 데이터 시각화 보완
- §02 "얼마나?"의 5.3명/연 같은 수치는 작은 차트(bar/donut)로 표현하면 spread 한 줄에 정보 압축.
- §01의 4 stat 카드는 지금 평면 카드 — 아이콘 1개씩 추가하면 인지 즉발.

### 6.5 폼 UX
- 4 필드 중 "신청 내용"이 textarea → 작성 부담.
- 옵션: "관심 항목" 체크박스로 변경 → 클릭 1~2회로 전송 가능하게.

## 7. SEO / 메타 / 공유

- `<title>`: `CLIPEDU — 원생매칭 플랫폼 포트폴리오` → "포트폴리오"는 외부 검색 키워드와 안 맞음.
- `<meta name="description">` 없음.
- Open Graph 태그(`og:title`, `og:image`, `og:description`) 없음 → SNS 공유 시 미리보기 깨짐.
- 페이지 H1이 hero 1개뿐. 일관됨(좋음).
- 한글 SEO 키워드(원생 매칭, 학원 마케팅, 학원 원생 모집 등) 노출이 본문 깊숙이만 있어 weak.

## 8. 접근성(A11y)

- 컬러 대비 — `--ink-2` (#5C6A85) on `--bg` (#EEF3FC): 약 5.4 :1 (AA pass for normal text). 보조 텍스트 사용에 적정.
- 아코디언 `aria-expanded`는 있음 ✓. 키보드 focus 스타일은 기본값 의존 → 커스텀 `:focus-visible` 권장.
- 지도 path의 `aria-label` 없음 — region 클릭은 마우스 전용.
- 이미지 `alt` 텍스트는 v2에서 처리됨(원장님과의 카카오톡 대화) ✓.

---

## 9. 성능

- Google Fonts 1회 로드(Space Grotesk + IBM Plex KR) — 권장 `font-display: swap`은 link tag로 제어 어려움. 임포트 부분에 명시되면 좋음.
- v2의 70 × 2 = 140장 이미지 — 첫 6장 eager + 나머지 lazy로 처리됨. 트랙 시작 전 ready class로 paused → OK.
- SVG 지도 path 다수 — 인라인이라 추가 요청 없음 ✓.

## 우선순위 — 이번에 잡으면 좋은 것 (Quick wins)

> "지금 한 시간 안에 적용 가능 + 임팩트 큰" 순.

| # | 작업 | 예상 영향 |
|---|---|---|
| 1 | **HERO에 CTA 2개 추가** | 첫 화면 전환율 직접 ↑ |
| 2 | **NAV에 "매칭 신청" CTA + 4개 anchor 메뉴** | 페이지 전체 전환율 ↑ |
| 3 | **숫자 일관성 수정** (1,200 vs 480, 60,000 vs 12,400) | 신뢰도 회복 |

## 마무리

이 페이지의 강점은 **시각 일관성**과 **데이터 풍부함**이다.
그러나 **첫 화면이 행동을 유도하지 않고**, **핵심 정보가 아코디언에 숨겨져 있으며**, **숫자가 서로 불일치**하면 그 강점이 전환으로 이어지지 못한다.

가장 먼저 손볼 곳은 다음 3가지로 압축됨:
1. **HERO + NAV에 CTA를 살린다** (전환 동선 확보)
2. **숫자 일관성·연도 표기를 정리한다** (신뢰)
3. **About의 3단락과 §02 어떻게? 4-step의 prose를 카드·bullet으로 분해한다** (텍스트 밀도 해소)

이 3가지만 적용해도 페이지 인상이 크게 달라질 것.
